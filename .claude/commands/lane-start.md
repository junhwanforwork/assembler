레인 착수를 실행한다 (**레인 세션 전용**). 워크트리 루트의 `PACKET.md`(오케스트레이터가 /wave-prep으로 하달)를 읽고 그 명세대로 즉시 착수, 완료하면 `REPORT.md`로 보고한다. 오케스트레이터가 REPORT.md를 자동 감지해 수거하므로 **사용자가 완료를 중계할 필요 없다.**

## Step 0 — 자기 위치 확인

cwd가 `.claude/worktrees/lane-N` 안이 **아니면**(= 메인 폴더 = 오케스트레이터 터미널) 즉시 중단 — "이 스킬은 레인 세션 전용이에요. 오케스트레이터는 /wave-prep·/wave-integrate를 쓰세요." 안내.

## Step 1 — 패킷 로드

워크트리 루트의 `PACKET.md`를 읽는다.

- **없으면 중단**: "착수 패킷이 없어요 — 오케스트레이터 세션에서 /wave-prep으로 하달해 주세요." (착수는 패킷 없이 하지 않는다 — 파일 소유 경계·/goal이 패킷에만 있다.)
- `REPORT.md`가 이미 있으면 = 이 웨이브 몫은 보고까지 끝난 상태 — 중단하고 "이 레인은 보고 완료, 다음 패킷 대기 중" 안내.
- 브랜치 확인(`git branch --show-current`): 패킷의 티켓 브랜치와 일치해야 한다. `lane-N-idle`이면 오케스트레이터가 브랜치 하달을 빠뜨린 것 — 중단·보고(레인이 임의로 브랜치를 만들지 않는다).
- 워크트리에 이미 커밋이 있으면(세션 재시작 등) 그 내용을 검증하고 이어서 진행한다 — 처음부터 다시 하지 않는다.
- **착수 신호**: 패킷을 읽었으면 즉시 `touch .lane-ack` (exclude 등재 — 오케스트레이터 워처가 "레인 착수"를 자동 감지한다).

## Step 2 — 제약 리마인드 + 착수

착수 전 아래를 한 번 출력한다:

1. 티켓 · 브랜치 · 파일 소유 범위(이 밖 수정 금지)
2. 금지 목록: push · merge · `memory/tickets.md` 수정 · DB 적용 · 테스트 삭제 · **크레덴셜 산출물 기재**(세션 id·키·토큰 — 로컬 .env.local에만) (로컬 커밋은 자유)
3. 패킷의 /goal (End · Verification · Constraints · Turn limit)

그다음 **패킷 본문을 그대로 이행한다** — 별도 확인 질문 없이 즉시 작업 시작. TDD 지시가 있으면 실패 테스트를 먼저 커밋(또는 실패 출력 캡처). 크로스체크는 레인에서 돌리지 않는다(오케스트레이터 몫).

## Step 3 — 완료 = REPORT.md 작성 (보고 창구)

작업 완료(또는 중단 조건 도달) 시, **마지막 행동으로** 워크트리 루트에 `REPORT.md`를 작성한다 — 이 파일의 생성이 곧 완료 신호다(오케스트레이터 워처가 감지). 작성 직전에 PACKET.md의 `## 추가 지시` 섹션을 재확인해 미이행분이 없는지 본다.

REPORT.md 고정 포맷:

```markdown
# REPORT — 레인 N · X차 웨이브 · <티켓>

- 상태: 완료 | 중단(사유)
- 브랜치/커밋: <ticket-slug> · <해시 목록 한 줄씩(해시 · 제목)>

## 한 일
(항목별 — 패킷 명세 대비 무엇을 했는지)

## 검증 출력
(tsc·lint·test·build 마지막 줄 수치 + e2e 수치. 실패가 있으면 실패 그대로 기록)

## 오케스트레이터 이월
(소유 밖 발견 · 판단 필요 · 편차와 그 사유. 없으면 "없음")

## 실수노트
(없으면 "없음")
```

REPORT.md 작성 후: 워크트리가 클린인지(`git status` — PACKET/REPORT는 exclude라 안 보임) 확인하고, "보고 완료 — 오케스트레이터가 자동 수거해요." 안내 후 **Step 4로 — 패킷 워처를 켜고 턴을 마친다.** `/checkout` 시 `memory/lane-logs/lane-N.md` append 관례는 유지(REPORT.md는 웨이브 후 삭제되는 창구, lane-log는 영구 기억 — 둘 다 쓴다).

## Step 4 — 패킷 워처 (다음 웨이브 자동 착수)

REPORT.md 작성 직후, **마지막 도구 호출로** 아래 워처를 `run_in_background`로 켜고 턴을 종료한다. 워처 종료 = 새 패킷(또는 추가 지시) 도착 신호이며, 백그라운드 태스크 완료로 이 세션이 자동 재개된다 — **사용자 타이핑 없이 다음 웨이브에 착수한다.**

```bash
# PACKET.md 내용이 바뀌면(새 웨이브 하달 or 추가 지시 append) 종료
h=$(md5 -q PACKET.md 2>/dev/null)
while true; do
  n=$(md5 -q PACKET.md 2>/dev/null)
  if [ -n "$n" ] && [ "$n" != "$h" ]; then echo "NEW PACKET"; exit 0; fi
  sleep 30
done
```

워처가 깨우면 = 새 하달이다: **Step 1부터 다시 수행한다**(새 PACKET.md를 읽고 즉시 착수 — 오케스트레이터가 새 하달 전에 이전 REPORT.md를 치우므로 Step 1의 "REPORT 있으면 중단" 가드에 걸리지 않는다). 이전 웨이브의 대화 맥락은 참고만 하고, 명세는 **오직 새 패킷**을 따른다. 컨텍스트가 무거워졌다고 판단되면(웨이브 3개 이상 연속 등) REPORT.md의 안내문에 "이 레인은 /clear 후 /lane-start 권장"을 남겨 오케스트레이터에 알린다.
