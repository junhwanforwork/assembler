편집 범위를 특정 디렉토리로 제한한다. 해당 디렉토리 외부 파일을 수정하기 전에 반드시 사용자에게 확인을 구한다.

## 사용법

```
/freeze src/components/builder
/freeze src/app/api/surveys
```

## 동작

지정한 디렉토리 경로를 절대 경로로 변환하고, 이 세션 동안:

- **허용**: 지정 디렉토리 내부 파일 읽기/편집/쓰기
- **경고**: 지정 디렉토리 **외부** 파일을 편집하려 할 때 사용자 확인 요청
- **항상 허용**: Read, Bash, Glob, Grep (편집이 아닌 조회)

## 경고 형식

```
⚠️  freeze 경계 외부 파일 편집 시도

파일: [파일 경로]
freeze 경계: [지정 디렉토리]

이 파일을 편집해도 괜찮습니까? (yes/no)
```

## 주의사항

- 이는 보안 경계가 아니라 실수 방지 도구
- `sed` 등 Bash 명령으로는 여전히 외부 파일 수정 가능
- 해제하려면 `/unfreeze` 실행

freeze 경계: $ARGUMENTS
