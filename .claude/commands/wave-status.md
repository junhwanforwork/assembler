cwd가 메인 폴더(`/Users/junhwanlim/Projects/assembler`)가 아니면(워크트리 = 작업 세션) 즉시 중단 — "이 스킬은 오케스트레이터 터미널 전용이에요" 안내. (읽기 전용이라 실해는 없지만, 작업 세션이 통합 판단을 하기 시작하는 것 자체를 막는다.)

## Step 1 — 대기열 현황

```bash
git worktree list
git branch --no-merged main        # 머지 대기 중인 레인 브랜치
```

레인별로: 브랜치 · `git log --oneline main..<브랜치>`(커밋 목록) · `git diff main...<브랜치> --stat`(변경 규모) · 워크트리 clean 여부(미커밋 잔여 = 아직 대기열 자격 없음).

## Step 2 — 겹침·충돌 예측 (워킹트리 안 건드림)

1. **레인 간 파일 겹침 매트릭스** — 레인 쌍마다 `git diff main...<A> --name-only`와 `git diff main...<B> --name-only`의 교집합. 겹침 = 머지 순서 후보(작은 쪽 뒤로).
2. **충돌 예측** — `git merge-tree --write-tree main <브랜치>` (트리만 계산, 체크아웃·커밋 없음). 순차 머지 시나리오까지 보려면 두 번째 레인부터는 첫 레인과의 merge-tree도 확인. 충돌 파일이 나오면 해당 hunk 요약.

## Step 3 — 레인별 게이트 스팟체크 (각 워크트리 안에서, main 오염 없음)

각 워크트리에서 `npx tsc --noEmit` + `npm test` 만 빠르게(전체 게이트는 통합 때). 실패 레인은 대기열 부적격으로 표기.

## Step 4 — 판정 리포트

| 레인 | 커밋 | 규모 | 스팟체크 | 충돌 예측 | 판정 |
|---|---|---|---|---|---|

- 판정: `준비됨` / `충돌 예상(해소 승인 필요)` / `부적격(사유)` + 권장 머지 순서.
