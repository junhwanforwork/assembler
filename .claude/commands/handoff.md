# handoff

현재 세션의 작업 상태를 요약해 다음 세션이 이어받을 수 있는 핸드오프 아티팩트를 생성한다.

**왜 필요한가:** Claude는 세션이 끝나면 기억을 잃는다. 다음 세션이 "어디까지 했지?"를 추론하는 데 시간을 낭비하거나, 이미 구현된 것을 "아직 안 됐다"고 판단하고 덮어쓰는 실수를 방지한다.

---

## 실행 절차

1. `git diff HEAD --stat` 실행 → 변경된 파일 목록 확인

2. `git log --oneline -5` 실행 → 최근 커밋 확인

3. `docs/specs/wip/` 디렉토리에 진행 중 티켓이 있으면 읽기

4. 아래 포맷으로 출력:

```
## 핸드오프 — {날짜 + 시간}

### 진행 중 티켓
HC-{id} — {title}
상태: {완료된 것 / 남은 것}

### 변경된 파일
{git diff --stat 출력}

### 핵심 결정 사항
- {구현 중 내린 비자명한 선택과 이유}
- {트레이드오프나 임시 방편으로 처리한 것}

### 다음 세션에서 할 일
1. {첫 번째 액션 — 구체적으로}
2. {두 번째 액션}

### 블로커 / 미결 질문
- {사용자 결정이 필요한 것}
- {외부 의존성}

### 다음 세션 시작 메시지 (복붙)
"HC-{id} 이어서 진행합니다. {상태 한 줄 요약}. 다음 할 일: {첫 번째 액션}"
```

5. 이 내용을 `docs/specs/wip/HC-{id}-handoff.md`에도 저장 (진행 중 티켓이 있는 경우)

---

## 언제 실행하는가

- 세션을 끝내기 전 (특히 구현이 중간에 멈출 때)
- `/eod` 이전에 진행 중 작업이 있을 때
- 다른 사람(또는 다른 Claude 인스턴스)에게 작업을 넘길 때
