레인 세션의 정체성과 상태를 복구한다 (**레인 세션 전용**). `/clear` 직후·새 터미널·상태가 헷갈릴 때 "내가 어느 레인이고 지금 뭘 해야 하는지"를 되찾는 진입점. **`/lane 2`처럼 번호를 주면 그 레인 워크트리로 직접 찾아간다.** `/lane-start`와 달리 **어떤 상태에서든 안전하게** 호출할 수 있다 — 상태를 판정해서 올바른 다음 행동으로 스스로 라우팅한다.

## Step 0 — 레인 결정 (인자 우선, 없으면 자동 감지)

**인자 N이 있으면 (`/lane 2`)** — 대상 = `.claude/worktrees/lane-N`:
- cwd가 이미 그 워크트리 안 → 그대로 진행.
- cwd가 메인 폴더 → `cd .claude/worktrees/lane-N`으로 이동(Bash cwd는 이후 호출에도 유지) 후 진행. **단, 이 창이 이 대화에서 오케스트레이터 역할을 해온 세션이면 중단** — 한 창 = 한 역할, 오케스트레이터 창을 레인으로 전용하지 않는다.
- cwd가 **다른** 레인 워크트리(lane-M, M≠N) 안 → 중단: "이 창은 레인 M이에요 — 레인 N은 그 레인 터미널에서 열어주세요." (한 폴더에 세션 둘 금지 — 작업 유실 원인.)
- `.claude/worktrees/lane-N`이 존재하지 않으면 중단: "레인 N 슬롯이 없어요 — 오케스트레이터에서 /wave-prep으로 만들어야 해요."

**인자가 없으면** — cwd에서 자동 감지:
- cwd가 `.claude/worktrees/lane-N` 안이면 그 레인. 메인 폴더면 중단: "여기는 오케스트레이터 터미널이에요 — 번호를 주면(`/lane 2`) 찾아갈 수 있어요."

결정되면 첫 출력으로 정체성 한 줄: **"여기는 레인 N — 브랜치 `<현재 브랜치>`."** 이후 모든 Step은 그 워크트리 안에서 수행한다.

## Step 1 — 제약 리마인드

레인 공통 제약을 한 번 출력한다:
- 이 워크트리 안에서만 작업(로컬 커밋 자유)
- 금지: push · merge · `memory/tickets.md` 수정 · DB 적용 · 테스트 삭제 · 크레덴셜 산출물 기재
- 크로스체크는 오케스트레이터 몫(레인에서 안 돌림) · 창구 = PACKET.md(받기)/REPORT.md(보고)

## Step 2 — 상태 판정 → 다음 행동

| 상태 | 판정 | 행동 |
|---|---|---|
| PACKET.md 있음 · REPORT.md 없음 | **작업 미완** | `/lane-start` 루틴으로 진입 — 기존 커밋이 있으면 검증 후 이어서(처음부터 다시 하지 않는다) |
| PACKET.md · REPORT.md 둘 다 있음 | **보고 완료 — 수거 대기** | 아무것도 다시 하지 않는다. Step 3 워처만 켜고 대기 |
| PACKET.md 없음 | **하달 대기** | Step 3 워처 켜고 대기 — 새 패킷이 오면 자동으로 깨어나 착수 |
| 브랜치가 `lane-N-idle`인데 PACKET.md 있음 | 하달 불일치 | 중단하고 사용자에게: "오케스트레이터에 브랜치 하달 누락을 알려주세요" |

상태와 다음 행동을 사용자에게 한 줄로 요약해 준다 (예: "보고까지 끝난 상태예요 — 다음 지시가 오면 자동으로 시작할게요.").

## Step 3 — 패킷 워처 재가동

`/clear`는 백그라운드 워처도 죽이므로, 대기 상태라면 워처를 다시 켜고 턴을 마친다(lane-start Step 4와 동일 — 워처 종료 = 새 하달 신호 = 세션 자동 재개):

```bash
h=$(md5 -q PACKET.md 2>/dev/null)
while true; do
  n=$(md5 -q PACKET.md 2>/dev/null)
  if [ -n "$n" ] && [ "$n" != "$h" ]; then echo "NEW PACKET"; exit 0; fi
  sleep 30
done
```

워처가 깨우면 = 새 하달: `/lane-start` Step 1부터 수행한다.
