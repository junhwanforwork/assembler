세션 마감 루틴을 실행한다. 코드 검사 → 티켓 동기화 → 실수·교훈 기록 → 규칙 반영 제안 → 세션 기록 → 정리. (/eod를 대체한다.)

실수 노트 파일: `/Users/junhwanlim/.claude/projects/-Users-junhwanlim-Projects-assembler/memory/mistakes.md`
티켓 파일: `memory/tickets.md` (**레포** — 단일 출처. 사용자 메모리의 옛 tickets.md는 잔재, 쓰지 말 것)
세션 로그 파일: `/Users/junhwanlim/.claude/projects/-Users-junhwanlim-Projects-assembler/memory/sessions.md`

## Step 1 — 코드 검사

이번 세션에서 코드 변경이 있을 때만 실행한다 (커맨드·문서만 바뀌었으면 스킵하고 요약에 "변경 없음" 표기).

```bash
npx tsc --noEmit 2>&1 | head -50
npm run build 2>&1 | head -50
```

소스(`src/**`) 변경이 있으면 추가로:

```bash
npm run lint 2>&1 | head -30
```

실패해도 여기서 멈추지 않는다 — 결과를 기록하고 Step 2의 티켓 상태 판정에 반영한다 (빌드 실패 = Done 불가).

## Step 2 — tickets.md 동기화

이번 세션에서 다룬 티켓을 `git log --oneline -10`, `git diff --stat`와 대조해 tickets.md의 체크박스·날짜·In Progress 상태가 실제와 맞는지 검증하고 정정한다.

Ticket Auto-Claim 규칙 기준:

- 코드 수정 완료 → verify 처리 (체크박스 + 날짜)
- 빌드/브라우저 검증 완료 → Done 이동
- 시작했지만 못 끝낸 티켓 → In Progress 유지
- Epic(`### 🎯 EPIC ASS-EXX`)은 체크박스가 없다 — 하위 티켓만 동기화하고, 한 epic의 하위가 전부 `[x]`면 epic 헤딩에 완료 표기(옵션).

## Step 3 — 실수 복기 → 실수 노트 저장

이번 세션의 대화·작업을 되짚어 실수를 식별한다:

- 잘못된 가정으로 시간을 낭비한 것
- 규칙(CLAUDE.md·`.claude/rules/*`) 위반으로 재작업한 것
- 빌드/타입 에러를 만든 코드 패턴
- 같은 에러를 2회 이상 반복한 것
- 사용자가 교정해 준 것
- (오케스트레이터 세션) **보고 문체·판단 오류**도 실수다 — 어려운 보고로 사용자가 되물은 것, 에이전트(크로스체크 등) 주장을 검증 없이 수용한 것

각 실수를 mistakes.md의 `## 실수 기록` 섹션에 추가한다. 형식:

```markdown
### [YYYY-MM-DD] 제목

- 실수: 무엇을 잘못했는지
- 원인: 왜 그랬는지
- 재발 방지: 다음에 어떻게 할지 (구체적 행동 규칙)
```

같은 실수가 이미 기록돼 있으면 중복 추가 금지 — 해당 항목에 `재발: N회 (마지막 YYYY-MM-DD)` 만 갱신한다. 재발 항목은 Step 5의 규칙 승격 1순위 후보다.

## Step 4 — 최적화·교훈 업데이트

이번 세션에서 더 나은 방법(도구·순서·패턴)을 발견했으면 mistakes.md의 `## 교훈 (더 나은 방법)` 섹션에 추가한다.

- 기존 교훈보다 나은 해법이면 옛 내용을 **대체**한다 (누적 금지 — 노트는 항상 현재 최선만 담는다)
- 기존 실수 항목의 "재발 방지"가 더 나은 방법으로 개선되면 그 항목도 갱신한다

## Step 5 — 규칙·스킬 승격 제안 (되새김)

승격 후보를 **세 소스**에서 스캔한다:

1. 이번 세션 실수·교훈 (Step 3·4 결과)
2. `mistakes.md`의 **재발 2회+ 항목** (마커 없는 것만)
3. `memory/lane-logs/*.md`(레포)의 **최근 2웨이브 실수노트** (마커 없는 것만)

판정 기준:
- 재발 2회+ = 승격 1순위.
- **유료 과금·보안·데이터 유실급은 1회 발생이어도 즉시 후보.**

승격 목적지 (규칙만이 아니라 **스킬도** 대상):
- 코드 규칙·금기·함정 → `CLAUDE.md` (Code Rules 또는 Known Issues & Caveats)
- 특정 경로에만 적용되는 규칙 → `.claude/rules/*.md` (paths frontmatter + `## Origin` 포함)
- 프로세스 실수(패킷 누락 항목·크로스체크 맹점·통합 절차 구멍) → `.claude/commands/*.md` (패킷 템플릿·크로스체크 지시문·통합 체크 보강)

제안 diff를 사용자에게 보여주고 **승인 후에만** 반영한다. 자동 반영 금지. 승격할 것이 없으면 "없음"으로 넘어간다.

**재제안 방지:** 반영이 끝난 항목은 원 기록(mistakes.md·lane-logs)에 `→ 승격됨(<대상 파일>, YYYY-MM-DD)` 마커를 남긴다. 마커 있는 항목은 다음 스캔에서 제외.

## Step 6 — 세션 기록 + 남은 티켓 맥락 저장

세션을 잇기 위한 핵심 단계. 티켓 작업이든 티켓 없는 작업이든 "어느 세션에서 무엇을 하고 있었는지"를 남긴다.

### 6-1. 가상 세션 명 부여 + sessions.md 기록

이번 세션에 가상 세션 명을 부여한다: `S-### · YYYY-MM-DD · 작업-슬러그`
(번호 = sessions.md 최상단 세션 번호 +1, 슬러그 = 이번 세션 작업을 요약한 한글/영문 2~4단어)

sessions.md **맨 위**(구분선 아래)에 엔트리를 prepend한다:

```markdown
## S-### · YYYY-MM-DD · 작업-슬러그

- 티켓: ASS-0XX (Done) · ASS-0YY (In Progress) / 없으면 "없음 (티켓 외 작업)"
- 티켓 외 작업: 티켓 없이 진행한 작업 — 무엇을, 어디까지 (없으면 생략)
- 상태: 완료 / 진행 중 (어디서 멈췄는지)
- 다음: 이어받을 때 첫 액션
```

한 세션에서 여러 갈래 작업을 했으면 한 엔트리 안에 모두 기록한다 (세션당 엔트리 1개).

**로그 정리 (세션이 쌓이는 것 방지):** 엔트리 추가 후, "완료" 상태 세션이 5개를 넘으면 가장 오래된 완료 세션들을 파일 하단 `## 아카이브` 섹션에 한 줄로 축약해 옮긴다 (`- S-003 · 날짜 · 슬러그 — 한 줄 요약`). **"진행 중" 세션은 절대 축약·삭제하지 않는다.**

### 6-2. In Progress 티켓 맥락 블록

In Progress 티켓이 있으면 tickets.md의 해당 티켓 항목 아래에 맥락 블록을 인라인으로 추가한다 (다음 세션이 바로 이어받을 수 있도록):

```markdown
- [~] ASS-0XX 제목
  > 맥락 (YYYY-MM-DD, S-###):
  > - 완료: ...
  > - 남음: ...
  > - 결정: 비자명한 선택과 이유
  > - 함정: 주의할 것
```

세션 명(S-###)을 반드시 포함한다 — 티켓에서 세션 로그로, 세션 로그에서 티켓으로 양방향 추적이 가능해야 한다. 이미 맥락 블록이 있으면 새로 추가하지 말고 갱신한다(세션 명도 최신으로). In Progress 티켓이 없으면 스킵.

### 작업 재개 시 (참고)

다음 세션에서 "하던 거 이어서" 요청이 오면: 티켓이 있으면 tickets.md 맥락 블록, 티켓이 없으면 sessions.md에서 슬러그·작업 내용으로 찾아 이어받는다.

## Step 7 — 미커밋 변경 + 프로세스 정리

```bash
git status --short
```

- 커밋 안 된 작업이 있으면 `/commit` 실행을 권유한다. **자동 커밋 금지.**
- assembler dev 서버가 떠 있으면 정리를 안내한다.
  ⚠️ `pkill -f "next-server"` 같은 광범위 매칭 절대 금지 — 반드시 `assembler` 경로 포함 정밀 매칭만 (CLAUDE.md Known Issues).

## 출력 — 세션 마감 요약

마지막에 아래 형식으로 요약한다:

```markdown
## 세션 마감 요약 — S-### · YYYY-MM-DD · 작업-슬러그

| 항목      | 결과                          |
| --------- | ----------------------------- |
| 타입 체크 | ✅ / ❌ (요지) / 변경 없음    |
| 빌드      | ✅ / ❌ (요지) / 변경 없음    |
| 티켓      | ASS-0XX Done · ASS-0YY 진행 중 |
| 실수 기록 | N건 (재발 M건)                |
| 교훈      | N건                           |
| 규칙 제안 | 있음(승인 대기) / 없음        |
| 미커밋    | N개 파일 / 없음               |
```
