## 핵심 설계 결정과 근거
1. **API 해석 = db-learning 대칭 복제** — 새 패턴을 만들지 않고 검증된 DB 해석 스택(evidence→
   iron_law→살균→보수 폴백→InsightCard)을 그대로 미러. 환각 방어가 이미 실전 검증된 구조라 안전.
2. **AI 추론은 asm_apis.summary가 아니라 신규 asm_api_notes 테이블** — summary는 syncApis가
   (product,method,endpoint) 멱등 upsert로 덮는 code-truth. 추론을 거기 두면 재싱크 때 증발
   (db_table_notes가 별도 테이블인 이유와 동일). 마이그레이션은 노트 테이블 RLS(부모 위임)를 미러.
3. **rate limit 신규 스코프 대신 "note" 재사용** — RPC 인자 가드 fail-open 함정(CLAUDE.md) 회피,
   11차 sync 재사용과 같은 판단. DB 노트와 카운터 공유는 승인된 절충.
4. **유료는 명시 트리거만** — API 해석 생성(anthropic)은 DataView "해석 만들기" 버튼 클릭 1곳에만.
   호버 툴팁·마운트는 GET(무료)뿐. ASM-048 자동 발사 제거 원칙 승계. 통합에서 코드 직접 확인.
5. **문서 오버레이 = OverlayPanel variant="window" 첫 소비** — 10차 프리미티브를 재사용해 신규
   서피스 0. docKind를 store로 승격해 좌 레일 하위행·중앙 뷰·오버레이가 한 선택 공유.
6. **레포 연동 진입 3갈래(target/pick/create)** — 프로젝트 선택/존재/없음으로 분기. 미선택인데
   프로젝트가 여럿이면 기존 선택 picker 제공(중복 생성 방지) — "안내 카피가 아니라 선택 수단".
