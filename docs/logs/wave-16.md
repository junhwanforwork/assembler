## 편성 판단 — 동작 계약 동결의 첫 적용
- 탐사가 store·EditorClient·editor.module.css `.body`를 다시 핫스팟으로 지목. Wave A 교훈(파일 안 겹쳐도 동작
  충돌) 적용: 파일 소유 + **동작 계약**(누가 무엇을 여닫나)을 함께 동결.
  - detailOverlayOpen 여닫기 = 레인1만(spec nonce + table trigger + TopBar). 레인2·3 미개입.
  - PromptDock=레인3만, SuggestionsCard=레인2만. store는 3레인이 다른 구획(레인1 제거 / 레인2 suggestions 추가 / 레인3 prefill 추가).
- 결과: 3레인 소스 겹침은 store 1파일뿐(제거+2 additive), 자동 머지 충돌 0. 동작 충돌도 0 — 계약 동결이 이번엔 통했다.
- 프롬프트 우/플로팅 위치(#9)는 우패널 삭제와 `.body` 그리드 한 줄을 다퉈 **다음으로 연기**(같은 레인 강제 → 과대). 정확한 판단.

## 핵심 설계 결정과 근거
- **상세 표면 = 플로팅 하나.** 우패널을 완전 삭제하고 TableInspector를 DetailOverlay로 흡수(inspected로 spec/table 분기).
  테이블 클릭도 이제 우패널 펴기가 아니라 플로팅 오픈. 숨은 도킹 인스펙터의 DOM 중복(Wave A에서 테스트 모호성의 근원)도 소멸.
- **AI 제안 = 아이템 3dot.** 상시 카드(창업자 #6 "뜬금없다")를 없애고 MoreVerticalIcon→Popover로. 유료 결과가 팝오버
  개폐로 유실되지 않게 **suggestions state를 store로 승격**(Popover는 열릴 때만 마운트). 유료 발사는 "분석하기" 1곳만.
- **Ask AI to edit = 프리필(전송 아님).** store `promptPrefill` 채널 → PromptDock가 `store.subscribe`로 구독해 input에
  채우고 포커스 후 비운다(1회성). 자동 전송 안 함 — 유료 발사는 사용자 확인 몫. effect 동기 setState는 react-hooks
  린트(cascading)에 걸려 **store.subscribe(외부 이벤트 반응) 패턴**으로 해소 — 렌더 중 setState 회피.
