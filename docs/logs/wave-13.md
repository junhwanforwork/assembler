## 한 줄 요약
사용자가 직접 쓰는 정책 문서(md 저작물)를 저장·편집하고, 본문에서 참조한 API·DB를 호버하면
해석/추천이 뜨게 조립. 레포의 기획 .md 문서도 읽어와 연결 미리보기에 표시. 창업자 결정 4건 반영.

## 편성 판단
- **왜 이 웨이브였나:** 아크2 잔여 실측(정책 문서 미착수·md 읽기 부분·git 쓰기 미착수·추천 부분)
  결과, **결정-게이트가 없는 것만** 편성했다. 창업자 결정 4건(2026-07-09): ① git 쓰기 = 앱 기능
  (사용자 GitHub 인증 설계) — 내 git과 무관, 보류 ② 배포 = 유보, 로컬 우선 ③ **P6(버전+diff) = 컷**
  ④ 개발자 추천 = 별도 기능 아님 → **정책 문서 안에서 참조 호버로 흡수**. 그래서 3레인 = 정책 문서
  BE/FE + md 읽기. git 쓰기·배포·추천-심화는 제외(결정/계정 필요).
- **3분해:** 레인1=BE(테이블·API) / 레인2=FE(에디터·호버) / 레인3=repo-extract md. 소유 배타(겹침 0).
  레인1→2 계약 동결(PolicyDoc). 레인3 독립.

## 핵심 설계 결정과 근거
1. **정책 문서 = product 소속 N행 자식 테이블** — 코드-진실(asm_apis처럼 product 전역)이 아니라
   저작물. asm_api_notes 자식 패턴 미러하되 **unique 없음**(부모당 여러 문서). 투사 문서(PRD 등)와
   별개 — 저장하는 유일한 문서 종류.
2. **개발자 추천을 정책 문서 호버로 흡수** — 창업자 지시. 별도 suggestions 재정향 대신, 정책 문서
   본문이 참조한 API·DB를 호버하면 12차 해석 카드(ApiNoteTip/TableRefTip)가 뜬다. 신규 유료 경로 0
   (호버=GET, 생성은 데이터 뷰 명시 버튼만). "안전하게"의 연장.
3. **레포 md는 읽기+미리보기까지만** — 영구 저장·정책 문서 임포트는 후속(계약 파급 최소). md는
   payload 아닌 report.docs(optional)로 — 싱크 파서 계약 보호. "문서만" 경계 = isMarkdownDocPath를
   좁게(시크릿 이름·코드 배제) + 기존 blocklist 위.
4. **rate limit 신규 스코프 대신 "sync" 재사용** — RPC fail-open 함정 회피(11·12차 선례).
