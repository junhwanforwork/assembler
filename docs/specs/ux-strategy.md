## UX 원칙 5

1. **여정에 편도 없음** — 모든 화면은 들어온 만큼 나갈 수 있다. 데드엔드·트랩 금지 (에디터 복귀, 빈 상태의 다음 행동).
2. **문 하나, 각도 여럿** — 같은 목적의 진입로는 1개(생성=Composer). 같은 모델을 보는 각도는 여럿이되, 각도마다 디렉토리가 못 보여주는 것을 보여줘야 존재 자격.
3. **AI는 제안하고, 사람이 그래프를 확정한다** — 그래프 변경은 전부 변경 계획 도크 승인 경유. 예외는 그래프 밖 레이어(DB 설명 노트)뿐.
4. **화면 언어 = 모델 언어** — "파일·문서·트리"가 아니라 "스펙·제품 구조·연결". 거짓 라벨(미구현 기능 약속) 0.
5. **일관성은 프리미티브로 강제** — 같은 인터랙션은 같은 컴포넌트(세그·툴팁·칩·모달). 방언 금지.

## 확정 구조 (질의응답 결과)

### 정보 구조 — 2층
```
프로젝트 ─ ChatGPT식 목록 ─→ 스펙(구 "파일"·워크스페이스)   ← 대시보드 + 에디터 스코프 드롭다운
스펙 안 ─ 모델 1개 ─→ 각도 5(문서·기능명세·플로우·와이어·데이터) + "제품 구조" 객체 트리
```
스펙 안에 "문서들"은 없다 — IDE식 파일 훑기 니즈는 객체 트리(요구사항→기능→페이지)가 흡수.

### 에디터 레이아웃
```
┌──────────────────────────────────┐
│ TopBar (로고=대시보드 복귀 · 스코프=스펙 전환) │
├──────┬───────────────┬───────────┤
│ 제품  │  CenterView   │ 우패널     │
│ 구조  │  (각도 본문)   │ 공용       │
│ (상시)│               │ 인스펙터   │  ← 전 뷰 선택 상세 단일 집 + suggestions·코멘트 합류
├──────┴───────────────┴───────────┤
│ 💬 AI 챗 도크 (접이식 · 변경 계획 시 자동 오픈) │
└──────────────────────────────────┘
```
- 챗 도크: claude.ai 자동 분할 패턴 — 변경 계획이 생기는 순간 열림. 대화하면서 트리·본문·상세 유지.
- 스펙 트리뷰: 숨김(코드 보존) → 4차에 N:M 교차 연결 그래프로 재설계 (레퍼런스 3사 모두 부재한 "연결 시각화" = 차별 자산).
