# 에디터 인터랙션 기능 정의서

> 출처: `design-prototypes/02-editor.html` (프로토타입 02)
> 목적: 에디터에서 **클릭·호버로 할 수 있는 모든 동작**을 빠짐없이 정의한다. 이미 배선된 동작과 아직 "정의 필요"인 빈칸(향후 기획)을 한 표에서 구분해, BE/FE 구현·기획 문서 확정의 기준 문서로 쓴다.
> `⚠협의` 행은 기대 결과가 애매해 **기획 문서와 맞춰가며 확정**해야 하는 행이다.
> **2026-07-02 확정 반영**: E·C 영역(S-003 협의) + A·D·F/G·J 영역(UX 전략 세션 — [ux-strategy.md](./ux-strategy.md)). 레이아웃 전제 변경: 우패널=전 뷰 공용 인스펙터, AI 챗=하단 도크(좌 토글 폐지), 스펙 트리뷰=숨김(4차 N:M 그래프 재설계).

## 상태 범례

| 상태 | 의미 |
|---|---|
| `배선됨` | 프로토타입 JS가 실제로 처리(시각 전환·토글 등). 단, 실제 데이터/저장 로직은 별도 |
| `부분` | 배선됐으나 실제 비즈니스 로직(저장·라우트·필터 적용)은 정의 필요 |
| `정의필요` | 핸들러 없음. 향후 기획·구현 필요 |
| `표시전용` | 클릭 대상 아님, 정보 표시만 |
| `⚠협의` (기대 결과 내) | 기대 결과가 애매해 사용자 확정 필요 |
| `확정` | 기대 결과 사용자 확정 — 구현 대기 |
| `구현됨` | 실제 앱 코드에 구현 완료 |
| `보류` | v1 범위 밖으로 확정(사유 명기) |

## 내보내기 설계 (확정 — 사용자 협의 완료)

내보내기에는 서로 다른 두 의도가 섞여 있어, 이를 분리해 정의한다.

### A. 개발로 — 구현 컨텍스트 *(MVP)*
- **단위:** 선택 기능 단위(기능명세서에서 고른 1~N개 기능).
- **내용:** PRD 요약 + 대상 기능명세·수용기준 + 유저플로우 경로 + **재사용/신규로 구분된 API·DB**(데이터 뷰의 `⎇ code`/`✶ mcp` 출처 활용) + 와이어프레임 참조.
- **핵심 가치:** 연결을 살린 채 코딩 에이전트(Claude Code·Cursor)에 전달 → *뭘 새로 만들고 뭘 재사용할지* 명시해 **환각 방지**.
- **형태:** 프롬프트 미리보기 + 복사/다운로드. (변경 계획 도크와 다름 — 도크는 *변경 적용*용)

### B. 협업도구로 — 동기화 *(Post-MVP)*
- **대상:** **Confluence**(조직 문서), **Figma**(연결을 주석으로 실어 보냄)가 우선. Notion은 MCP가 연결돼 있으나 후순위.
- **설계 방향:** "내보내기(정적 스냅샷)"보다 **"동기화"**로 설계 권장 — 정적 스냅샷은 원본과 어긋난다. 이 경우 내보내기 메뉴가 아니라 통합 설정에 가까워질 수 있음.
- **MVP 처리:** 메뉴엔 "곧" 비활성으로 노출해 방향만 제시.

### 보조 / 제외
- **Markdown 묶음:** 보조(이식·백업). 명세가 이미 .md라 부담 적음.
- **git 레포 내보내기:** 제외.
