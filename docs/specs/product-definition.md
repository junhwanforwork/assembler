## 1. 제품 정의

**한 문장:** Assembler는 **AI 코딩(Claude Code 등)이 참조하는 "md 두뇌"를, 연결된 스펙 그래프에
근거해 만들고 관리하는 도구**다 — 비개발자·기획자가 안전하게 개발에 접근하게 하는 기획 보조.

**두 축을 하나로(핵심):**
- **① 스펙 그래프(설계)** — 요구사항·기능·화면·플로우가 서로 연결되고, API·DB(코드 진실)에 물린
  구조. "무엇을 만드나"의 머리. 여기서 기획하고 편집한다.
- **② md 파일(두뇌·산출)** — repo에 사는 rule·agent·정책·문서·참조 md. **AI 코딩이 읽는 참조 레이어.**
  보기·편집·생성·**main push**가 1급 기능. **최종 목적 = 실제 코드의 정책·사실을 md 참조 파일로 옮겨,
  AI 코딩이 그걸 보고 안전하게 짓게 한다.**

**한 바퀴(루프):** 그래프에서 기획 → md 생성/편집 → **main push** → AI 코딩이 md 참조 → 코드 바뀜
→ 코드 진실(API·DB) 갱신 → 그래프에 반영. **"스펙은 한 곳(그래프), 산출은 md, 변경은 한 번."**

**해결하는 문제 — 기획과 개발 사이의 간극:**
- 비개발자는 레포를 두려워한다(뭘 건드리면 안 되는지, API·DB가 뭔지 모른다).
- 개발자는 기획 문서를 불신한다(코드와 안 맞고 바뀌어도 전달 안 됨).
- 기존 도구는 문서에서 끝난다 — 문서 속 "API"와 실제 코드의 API가 남남이다.

**해자(우리의 답):** 문서·md 속 모든 API·테이블·기능이 **실체(코드-진실)와 연결**되고, 호버하면
**비개발자 말로 해석**되며, **md로 뽑아 main에 올려 AI 코딩이 참조**한다. 나아가 **"이 기능이 어느
코드 파일에 영향·어떻게 개선"** 까지 짚는다(추출기가 소스 파일 경로를 알기 때문).

**이번에 하지 않는 것 (명시적 후퇴·컷):**
- 코드 **직접 수정**은 안 한다 — API·DB "개선"은 드물게 개발자 수정 요청으로(제품은 기획 보조).
- 와이어프레임·UI 요소 — later phase(휴면, 데이터는 보존).
- 배포(M2)·버전+diff(P6) — 유보/컷(2026-07-09).

## 2. 사용자와 핵심 여정

**주 사용자 = 비개발자/기획자**(창업자·바이브 코더): 레포가 있거나 아이디어만 있다.
**부 사용자 = 개발자**: md를 받아 구현하고, 역으로 개선을 준다.

**여정 A — 비개발자 (핵심):**
1. **가져오기·연결**: 레포 연결(폴더·깃) → 코드진실(API·DB) 추출 + 기존 md(rule·agent·기획 md)
   가져오기. (또는 아이디어로 시작해 그래프 생성.)
2. **이해**: API·테이블·참조를 비개발자 말로 해석받는다(호버).
3. **기획(설계)**: 기능 중심으로 요구사항·기능·화면·플로우를 만들고 고친다(챗 변경 → 한글 diff →
   영향 범위).
4. **문서·md 만들기**: 자동 문서(PRD 등)는 그래프에서 나오고, 정책·참조 md는 직접 쓴다.
5. **전달**: md를 **main에 올린다(push)** → AI 코딩(또는 개발자)이 그 md를 참조해 짓는다.

**여정 B — 개발자:**
1. 전달받은 md(기술 명세·정책·참조)를 본다.
2. 구현 컨텍스트를 내보낸다(코딩 에이전트용).
3. **개선을 준다**: "이 기능/화면에 이 API 개선이 필요" — 드물게, 기획 보조.

**북극성 지표(유지):** 변경 1건당 수정 지점 수 = 1 · 변경 반영 소요 시간.

## 3. 기능 정의 (Key Feature → 세부)

> 조작(신규/수정/삭제/main올리기)은 §4-4 조작표가 단일 출처. 모든 산출물은 연결(고립 금지).

### KF1 · 시작·연결 — "가져와서 시작한다"
- 아이디어로 시작 → 스펙 그래프 생성. / 레포 연결(폴더·깃) → 코드진실 자동 추출.
- 기존 md 가져오기(rule·agent·CLAUDE.md·기획 md). / 코드진실에 **소스 파일 경로**까지 보존.

### KF2 · 설계(스펙 그래프) — 기능이 허브, 여기서 편집
- 기능·요구사항·세부기능·화면·플로우 = 신규/수정/삭제. 기능이 요구사항·화면·API·DB를 연결.
- 무결성: 끊어진 참조는 저장에서 거부(409). (화면 안=와이어는 later.)

### KF3 · 문서 — 자동(투사) + 자유 저작(한 종류)
- 자동 문서: PRD·기술명세·데이터사전 = 그래프 절단면(**읽기전용**). "원본 고치러 가기" + "md로 내보내 push".
- 자유 저작 문서(정책=템플릿): add/수정/삭제 + 그래프 링크. 오버레이 창. 본문 참조 호버 해석.

### KF4 · 해석 — 비개발자 말로, 환각 차단
- DB·API·문서 참조 호버 설명(구조=사실/설명=추론 분리). "어떻게 개선하면 좋을까" 추천.

### KF5 · 코드 진실 — 읽기, 근거, 파일까지
- API 목록/상세·DB 관계도. 기능→API/DB→**소스 파일 경로**("이 기능이 건드리는 파일"). 개발자 개선 요청(드묾).

### KF6 · 변경 — "한 번 고치면 다 반영"
- 챗 변경 계획 → 한글 diff 승인 → 적용 → 영향 범위 자동 표시(+ **어느 파일**) → 활동 기록.

### KF7 · 전달·md 두뇌 — repo에 올리고 AI 코딩이 참조
- 구현 컨텍스트 내보내기. **문서·정책·규칙 md → main push**(git 쓰기).
- **참조 md 생성(코드진실 → md) ← 최종 목적.** 규칙·에이전트 md 관리(view/edit/신규).

## 4. IA — 정보 구조

### 4-1. 화면 구조 (에디터 첫 뎁스 = 설계 / 문서·md / 코드진실)
```
대시보드 — 제품 목록 · 시작(아이디어 / 레포 연결 / md 가져오기)

에디터
├─ 좌 레일
│   ■ 설계 (원본 그래프 · 편집)
│      ├ 기능        ← 홈·초기 뷰(현 기능명세서 재프레임 — 요구사항·기능·세부기능). 기능이 허브
│      └ 사용자 플로우  ← 화면 간 흐름.  (later) 화면/스토리보드 = 와이어 부활
│   ■ 문서·md (repo에 올라가는 것 · AI 코딩 참조 · main push)
│      ├ 자동 문서(투사·읽기): PRD · 기술명세 · 데이터사전
│      ├ 문서(자유 저작): 정책 등 — add/수정/삭제
│      ├ 규칙·에이전트 md — 가져오기·편집
│      └ 참조 md — 코드진실을 옮긴 것
│   ■ 코드 진실
│      └ DB · API (+ 소스 파일 경로)
├─ 중앙: 선택한 뷰
├─ 우 패널: 인스펙터(상세 + 해석 카드)
├─ 하단: 챗 도크(변경 계획·승인)
├─ 오버레이: 문서 창
└─ (아크2) git 상태 바 / "main에 올리기"
```
- **기능명세서는 "문서" 아님 → "설계(원본)".** 문서·md 항목엔 **출처 뱃지**(자동읽기/저작/코드진실)·
  **연결 뱃지**(무엇에 연결)·**"main 올리기"** 를 붙인다.

### 4-3. 연결 지도
```
요구사항 ←─ 기능 ─→ 화면 ←─ 플로우
              ├─→ API (코드진실 · 소스파일)
              └─→ DB  (코드진실 · 소스파일)
문서(자유·정책) ─링크→ 기능·API·테이블 (복붙 금지)
투사 문서 3종 = 위 그래프의 절단면(자체 데이터 0)
md(문서·정책·규칙·참조) ── main push ──→ repo ──→ AI 코딩이 참조
```

### 4-4. 조작표 (출처가 조작을 정한다 — 단일 출처)
| 항목 | 출처 | 신규 | 수정 | 삭제 | main 올리기 |
|---|---|---|---|---|---|
| 요구사항·기능·세부기능·화면·플로우 | 저작(그래프) | O | O | O | (문서로) |
| PRD·기술명세·데이터사전 | 투사(자동) | X | X(원본 고침) | X | O(md로) |
| 문서(자유 저작, 정책 포함) | 저작 md | O | O | O | O |
| 규칙·에이전트 md | 가져오기+저작 | O | O | O | O |
| 참조 md | 코드→md | O | △ | O | O |
| API·DB | 코드진실 | X | X(개발자 요청) | X | (참조 대상) |

- 편집 가능=인라인 편집+"＋새로 만들기"+저장(삭제=확인 다이얼로그). 투사=읽기+원본 링크+내보내기.
  코드진실=읽기+개선 요청. 저작/생성물마다 연결 뱃지(고립 금지).

## 5. 단계 (웨이브)
